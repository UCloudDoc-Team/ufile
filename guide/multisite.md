# 存储桶复制

US3 提供存储桶复制功能，您可以为源存储空间（Bucket）绑定一个目标存储空间（Bucket），对文件进行自动同步。

存储桶复制是针对两个存储空间之间的自动、异步复制能力，会将文件的创建、更新和删除等操作从源存储空间复制到目标存储空间。存储桶复制既支持跨地域复制，也支持同地域存储空间之间的复制。

存储桶复制功能可用于数据容灾、数据备份、业务分发以及数据迁移等场景。目标存储空间中的文件是源存储空间中文件的完整副本，具有相同的文件名和内容。

## 使用场景

* 合规性要求：您的数据在合规性上必须要求跨一定距离保存一份副本。通过存储桶复制功能，可以将数据复制到其他地域的 US3 数据中心，以满足相关合规要求。

* 最大限度减少延迟：您在多个地理位置存在业务需求时，可在更靠近用户的 US3 数据中心中维护文件副本，以降低访问延迟。

* 数据备份与容灾：您对数据的安全性和可用性有极高要求时，可通过存储桶复制在不同存储空间中保留副本，以便在极端情况下快速启用备份数据。

* 数据复制：由于业务原因，需要将您的数据从一个 US3 存储空间迁移到另一个存储空间。

* 操作协同：您在不同地域或同一地域的多个环境中都需要使用相同数据时，可通过存储桶复制同时维护多个存储空间中的文件副本。

## 注意事项

1. 对于处于同步状态的两个存储空间，由于您可以同时操作这两个存储空间，因此从源存储空间复制过去的文件可能会覆盖目标存储空间中的同名文件，覆盖文件的操作不可逆，请根据您的需求合理设置存储桶复制规则，避免重要数据丢失。

2. 由于存储桶复制采用异步复制，数据复制到目标 Bucket 需要一定时间，通常为几分钟到几小时不等，具体取决于数据量大小。

3. 镜像回源与存储桶复制两个功能不能同时开启。

4. 如存储桶配置了生命周期规则，则生命周期配置仅对当前存储空间生效，开启存储桶复制的存储空间之间不会同步删除文件。

5. 存储桶复制相关费用：

- 流量费用：使用存储桶复制功能将源存储空间的数据同步复制到目标存储空间时，若涉及跨地域复制，可能产生源端外网流出费用。流量费用请参见[产品价格](https://docs.ucloud.cn/ufile/bill/billing)。
- 请求费用：US3 会计算请求次数并收取请求费用，包含源端的拉列表和下载请求以及目标端的上传请求。请求费用请参见[产品价格](https://docs.ucloud.cn/ufile/bill/billing)。

## 设置存储桶复制规则

选中对应空间，进入管理界面，选择存储桶复制功能。

![](/images/跨区域复制3.png)

点击添加规则按钮，弹出添加存储桶复制规则界面。

![](/images/跨区域复制4.png)

1. 目标地域：用户可选择复制目标存储空间所在地域。若进行跨地域复制，请选择目标存储空间所在地域；若目标存储空间与源存储空间位于同一地域，则选择相同地域即可。

2. 目标存储空间：选择开启数据同步的目标存储空间。一条存储桶复制规则仅支持配置一个目标存储空间。单个存储空间最多可绑定 100 条同步规则，如需更高上限，请联系技术支持。

3. 历史文件同步：开启后，会将当前存储空间内指定同步范围的历史文件同步到目标存储空间；不开启则仅同步规则生效后的新增文件。

4. 数据同步策略：针对增量数据，可选择同步新增、删除、修改操作，或仅同步新增、修改操作。

5. 同步范围：可选择整个存储空间或指定前缀。若配置前缀为 `test`，则会匹配所有以 `test` 开头的对象 Key，例如 `test/a.txt`、`test.txt`、`test-1/demo.jpg` 均会被匹配；若仅希望匹配 `test` 目录下的文件，建议将前缀配置为 `test/`。

存储桶复制规则添加完成后默认生效，具体状态可见状态栏，您可以通过操作栏下的其他按钮进行启用、禁用以及删除操作。

## 备注

1. 历史文件同步仅针对该规则启用时刻之前已存在的文件生效。若规则在启用后被禁用，再次启用时仅会继续处理增量同步，不会重新触发历史文件同步。如需再次执行历史文件同步，请删除原有规则后重新创建规则。

2. 关于增量文件同步，规则启用后到配置真正生效之间可能存在最长 5 分钟的延迟，建议不要在启用后立即触发同步相关操作。

3. 存储桶复制功能支持数据实时同步，对于数据的增加、删除、修改能够实时监控并同步到目标存储空间。对于 2MB 文件，通常可做到分钟级同步，保证两边数据的最终一致性，但不保证中间状态的数据一致。

4. 存储桶复制目前不支持国内和国外地域之间的复制。如有需求，请联系技术支持。

5. 归档类型文件不支持复制。
